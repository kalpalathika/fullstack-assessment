# StackShop ‚Äî Bug Fix Assessment

## Overview

I reviewed the StackShop application end-to-end and identified **9 bugs** spanning security, functionality, UX, and code quality. Below is a summary of each issue, the fix applied, and the reasoning behind my approach.

---

## Bug Fixes

### 1. Missing Amazon CDN Hostname
**File:** `next.config.ts`

**Issue:** Only `m.media-amazon.com` was whitelisted in `remotePatterns`. Several products use `images-na.ssl-images-amazon.com`, causing Next.js to throw a runtime image error for entire categories (e.g. Desktops).

**Fix:** Added the second hostname to `remotePatterns`.

---

### 2. `imageUrls` Undefined Crash
**File:** `app/page.tsx`

**Issue:** `product.imageUrls[0]` throws a `TypeError` when `imageUrls` is absent from a product in the dataset.

**Fix:** Changed to `product.imageUrls?.[0]` using optional chaining.

---

### 3. Stale Page Metadata
**File:** `app/layout.tsx`

**Issue:** Title and description were still the default Next.js boilerplate ‚Äî "Create Next App" / "Generated by create next app".

**Fix:** Updated to "StackShop" with a relevant description.

---

### 4. `retailPrice` Not Displayed
**Files:** `lib/products.ts`, `app/page.tsx`

**Issue:** `retailPrice` exists on every product in the dataset but was missing from the `Product` interface entirely, so it was never typed or rendered anywhere in the UI.

**Fix:** Added `retailPrice?: number` to the `Product` interface and displayed it formatted as currency on both the product card and detail page. Also fixed inconsistent card button alignment using `flex flex-col` + `flex-1` on the card layout.

---

### 5. Subcategory Filter Ignored Selected Category
**File:** `app/page.tsx`

**Issue:** The frontend called `/api/subcategories` without passing a `?category=` parameter. The API correctly supports filtering by category ‚Äî the frontend just never sent it. Result: all subcategories from every category appeared in the dropdown regardless of selection.

**Fix:** Passed `?category=${encodeURIComponent(selectedCategory)}` in the fetch. Also wired up the `total` field from the API response to show "Showing X of Y products" instead of a hardcoded 20.

---

### 6. Clear Filters Didn't Reset Category Dropdown Visually
**File:** `app/page.tsx`

**Issue:** Category state was initialized as `undefined`. Radix UI's `<Select>` with `value={undefined}` becomes uncontrolled and won't visually reset to its placeholder when cleared.

**Fix:** Initialized `selectedCategory` as `""` (empty string) so the Select stays controlled and resets correctly.

---

### 7. Search Fired API Call on Every Keystroke
**File:** `app/page.tsx`

**Issue:** `search` state updated on every `onChange` event, triggering a full API call per character typed. Typing "gaming keyboard" fired 14 requests.

**Fix:** Added a 300ms debounce via a `debouncedSearch` state ‚Äî the products fetch only runs after the user pauses typing.

---

### 8. No Error Handling on Fetch Calls
**File:** `app/page.tsx`

**Issue:** All `fetch()` calls had no error handling. On network failure, `loading` stayed `true` forever with no user feedback.

**Fix:** Wrapped the products fetch in `.catch()` with an `error` state and a user-facing error message rendered in place of the product grid.

---

### 9. Entire Product JSON Serialized into URL (Security)
**Files:** `app/page.tsx`, `app/product/[sku]/page.tsx` (new), `app/product/page.tsx` (removed)

**Issue:** The full product object was `JSON.stringify`'d into the URL query parameter and `JSON.parse`'d on the detail page without any validation. This meant:
- Anyone could craft a URL with arbitrary product data and render it on the domain
- The list-page `Product` type was narrower than the full shape ‚Äî `featureBullets` was missing, so feature bullets never rendered
- URLs were 500‚Äì1000+ characters and not shareable

**Fix:** Replaced `/product?product=...` with a proper `/product/[sku]` dynamic route. The URL now contains only the SKU. The page is a **server component** that calls `productService.getById(sku)` directly ‚Äî no client-side JSON parsing, no user-controlled data. Invalid SKUs return Next.js's built-in 404. The interactive image gallery is extracted into a small `ImageGallery` client component.

---

## Summary

1. Missing Amazon CDN hostname causes image crashes on multiple categories (High)
2. Accessing `imageUrls[0]` without null check crashes product list (High)
3. Page title and description left as Next.js boilerplate (Low)
4. `retailPrice` field missing from type and never shown in UI (Medium)
5. Subcategory dropdown shows all subcategories regardless of selected category (High)
6. Clear Filters button doesn't visually reset the category dropdown (Medium)
7. Search input fires an API call on every single keystroke (Medium)
8. Failed fetch calls silently hang with no user-facing error (Medium)
9. Full product JSON passed through URL allowing arbitrary data injection (High)

---

## Future Enhancements

1. **Pagination** ‚Äî The API supports `limit` and `offset` but the UI hardcodes `limit=20` with no way to load more. Would add pagination or infinite scroll with page state reflected in the URL.
2. **Database Integration** ‚Äî Products are currently loaded from a static JSON file. In production this would be replaced with a database (e.g. PostgreSQL via Prisma) with indexed queries for efficient search and filtering.
3. **API Input Validation** ‚Äî Route handlers accept query parameters without sanitization. Would add schema validation (e.g. Zod) at the API boundary to reject malformed or oversized inputs.
4. **Unit & Integration Tests** ‚Äî No tests exist in the codebase. Would add component tests with React Testing Library and API route tests with Jest to catch regressions on the bugs fixed here.
